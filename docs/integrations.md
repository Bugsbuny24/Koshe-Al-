# Koschei — External Integration Layer

> Generated by the layered integration merge. All integrations default to `enabled: false`.
> No integration should be activated until its environment variables are configured and a staging soak test is completed.

---

## Integration Command (Origin)

```bash
copilot integrate \
  --base-repo ./koschei \
  --add https://github.com/langgenius/dify \
  --add https://github.com/triggerdotdev/trigger.dev \
  --add https://github.com/appydave-templates/starter-nextjs-convex-ai \
  --add https://github.com/mem0ai/mem0 \
  --add https://github.com/bytedance/deer-flow \
  --add https://github.com/ruvnet/RuView \
  --add https://github.com/koala73/worldmonitor \
  --add https://github.com/obra/superpowers \
  --preserve-structure \
  --merge-strategy layered \
  --validate-build \
  --generate-docs
```

**Flags applied:**
- `--preserve-structure` — Koschei's existing module hierarchy is unchanged.
- `--merge-strategy layered` — Each integration is added as a new layer; it does not replace existing modules.
- `--validate-build` — TypeScript build must pass after each integration is registered.
- `--generate-docs` — This file and the updated module-map are auto-generated.

---

## Architecture Position

Integrations sit **above** the existing V1–V20 module layers. They provide external capabilities that power specific Koschei modules, but they do not own routes or business logic — they are adapters.

## Using Integration Flags

Integration flags follow the same pattern as Koschei module flags:

```typescript
import { isFeatureEnabled } from '@/lib/features/flags';

if (isFeatureEnabled('INT_DIFY_WORKFLOW')) {
  // use Dify workflow client
}
```

Reading integration metadata:

```typescript
import { featureFlags } from '@/lib/features/flags';

const flag = featureFlags['INT_DIFY_WORKFLOW'];
console.log(flag.label);       // "Dify Workflow Integration"
console.log(flag.description); // full description
```

Querying the registry:

```typescript
import { getIntegrationsForCapability, getActiveIntegrations } from '@/lib/integrations';

// which integrations power the Production Engine?
const productionIntegrations = getIntegrationsForCapability('production-v4');

// all non-deprecated integrations
const active = getActiveIntegrations();
```

## Activation Checklist

Before enabling any integration flag in production:

- [ ] All required environment variables are set in the deployment environment
- [ ] A staging environment test has been run for at least 48 hours
- [ ] Any privacy implications (e.g., Mem0) have been reviewed and consent flows added
- [ ] Cost/rate-limit implications have been assessed
- [ ] The corresponding Koschei module flag (e.g., `V4_PRODUCTION_ENGINE`) is also enabled
